# 002 — npm publish + binarios standalone

## Objetivo

Que `npm install -g memlink`, `bun add -g memlink`, `pnpm add -g memlink` funcionen (publicando en npm registry),
y que `curl rblez.com/memlink/install.sh | bash` descargue un binario standalone precompilado desde GitHub Releases.

---

## 1. `package.json`

### `"files"` — incluir solo runtime

```json
"files": ["dist", "README.md", "LICENSE"]
```

### `"exports"` — apuntar a `dist/` en vez de `.ts`

```json
"exports": {
  ".": {
    "import": "./dist/cli/index.js",
    "types": "./dist/cli/index.js"
  },
  "./server": {
    "import": "./dist/server/index.js",
    "types": "./dist/server/index.js"
  }
}
```

### `"prepublishOnly"` — build automático antes de publish

```json
"prepublishOnly": "npm run build"
```

`npm run build` usa `bun build` → genera `dist/cli/index.js` + `dist/server/index.js`.

### `"publishConfig"` — acceso público

```json
"publishConfig": {
  "access": "public"
}
```

### Limpiar dependencies mal ubicadas

- `@eslint/js` → mover a `devDependencies`
- `typescript-eslint` → mover a `devDependencies`
- Eliminar `@typescript-eslint/eslint-plugin` y `@typescript-eslint/parser` de `devDependencies` (ya los incluye `typescript-eslint`)

## 2. Release workflow (`.github/workflows/release.yml`)

Agregar job `publish-npm` después de `build`:

```yaml
publish-npm:
  name: Publish to npm
  needs: build
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: oven-sh/setup-bun@v1
      with:
        bun-version: latest
    - run: bun install
    - run: npm run build        # genera dist/
    - run: npm publish
      env:
        NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

Requisito: agregar `secrets.NPM_TOKEN` en settings del repo.

## 3. `install.sh` — reescribir para binarios standalone

Actualmente intenta `bun add -g memlink` / `npm install -g memlink` (falla porque no está publicado).

### Nueva lógica

```
1. Detectar OS + arquitectura (ya existe)
2. Armar nombre: memlink-{os}-{arch}   (ej: memlink-linux-x64)
3. Descargar de GitHub Releases:
     https://github.com/rblez/memlink/releases/latest/download/{nombre}
4. chmod +x
5. Instalar a ~/.local/bin/ (fallback /usr/local/bin/)
6. Ejecutar memlink init
7. Reportar éxito o error
```

### Cambios clave

| Antes | Después |
|-------|---------|
| Hardcodea `MEMLINK_VERSION="0.4.0"` | Obtener versión via GitHub API (`/repos/rblez/memlink/releases/latest`) |
| Instala Bun si no existe (pesado) | No necesita Bun — binario precompilado |
| `bun add -g memlink` | `curl -L` del binario + `chmod +x` + `mv` a PATH |
| Sin cleanup | `rm` del binario descargado tras instalación |

## 4. Resumen: 3 rutas de instalación

| Método | Origen | Requisito previo |
|--------|--------|------------------|
| `npm install -g memlink` | npm registry | Node.js + npm |
| `bun add -g memlink` | npm registry | Bun |
| `pnpm add -g memlink` | npm registry | pnpm |
| `curl rblez.com/memlink/install.sh \| bash` | GitHub Releases | Ninguno (binario standalone) |
